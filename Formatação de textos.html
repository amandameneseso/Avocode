Semântica e como tags morrem

"Semântica é o significado dos vocábulos, por oposição à sua forma." Em HTML5, vemos de forma bastante evidente a presença do chamado HTML semântico ou tags semânticas, logo suas tags tentam levar um significado embutido muito forte. Você quer que um endereço apareça na forma de um texto sublinhado? Use CSS para configurar isso, a HTML serve para dar sentido ao conteúdo.

Existem tags que ainda funcionam hoje em dia, mas estão prestes a serem consideradas obsoletas. Isso acontece porque a linguagem evolui, e nesse processo algumas tags param de fazer sentido pois existem outras bem melhores e que fazem mais sentido. Clique aqui para consultar os elementos obsoletos na HTML5.

Principais formatações

1.1 Negrito/destaque

Termo em negrito usando a tag <strong>

1.2 Itálico/ênfase

Termo em itálico usando a tag <em>

1.3 Texto marcado

Texto marcado usando a tag <mark>

1.4 Texto marcado com cor diferente utilizando CSS

Texto marcado com cor diferente usando "style" dentro da tag <mark>. Exemplo:

<mark style="background-color: lightblue;">Texto marcado com cor diferente</mark>

1.5 Texto grande e pequeno

A tag <big> (grande) está obsoleta mas ainda funciona. Texto pequeno usando a tag <small>

Texto deletado usando a tag <del>

1.7 Texto inserido/sublinhado

Texto sublinhado usando a tag <ins>

1.8 Texto sobrescrito

Para escrever coisas do tipo x2 usando a tag <sup>

1.9 Texto subscrito

Para escrever coisas como H2O usando a tag <sub>

1.10 Trecho de código

A tag <code> delimita um código. A principal vantagem no uso dessa tag é o valor semântico que ela representa, indicando ao navegador que se trata de um código de computador. Porém, existe também um efeito visual, pois as letras ficam no modo mono-espaçadas. Exemplo:

<code>document.getElementById('teste')</code>

1.10.1 Tag <pre>

Utilizada para representar texto pré-formatado. Um texto dentro desse elemento é tipicamente exibido em uma fonte não proporcional da mesma maneira em que o texto original foi disposto no arquivo. Espaços em branco são mantidos no texto da mesma forma em que este foi digitado. Exemplo:

                
            <pre>
                <code>
                    num = int(input('Digite um número'))
                    if num % 2 == 0:
                        print(f'O número {num} é PAR')
                    else:
                        print(f'O número {num} é ÍMPAR')
                    print('Fim do programa')
                </code>
            </pre>

1.11 Citações simples

A tag <q> adiciona aspas. Usada quando queremos uma citação no meio de um parágrafo. Exemplo:

Como diria o pai de um amigo: <q>o computador é um burro muito rápido.</q>

1.11.1 Citações completas

A tag <blockquote> é usada para citações mais longas com um parágrafo só para si. O texto ganha um recuo automaticamente. Podemos também colocar um link para o texto original, usando o parâmetro cite dentro da tag. Exemplo:

<p>Segundo Jeff Noble, no seu livro HTML para Leigos:</p>
<blockquote cite="https://books.google.com.br/books?id=E8ZtDwAAQBAJ&pg=PA17&dq=HTML&hl=pt-BR&sa=X&ved=0ahUKEwjH5-PevNboAhVNIbkGHcoxCOgQ6AEIOjAC#v=onepage&q=HTML&f=false">
A diferença entre elementos inline e um bloco de texto é importante. Os elementos HTML neste capítulo descrevem os blocos de texto.
</blockquote>

1.12 Abreviações

A tag <abbr> define uma abreviação e opcionalmente fornece uma descrição completa para ela. Se presente, o atributo title deve conter a descrição completa e apenas ela. Exemplo:

<abbr title="Cascading Style Sheets">CSS</abbr> é a linguagem que descreve o estilo de um documento <abbr title="HyperText Markup Language">HTML</abbr>.

CSS é a linguagem que descreve o estilo de um documento HTML.
