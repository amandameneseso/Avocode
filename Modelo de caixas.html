Modelo de caixas

Os conceitos do Modelo de Caixas (Box Model) são aplicáveis ao HTML e ao CSS. Se trata de conjuntos que irão facilitar muito na hora de organizarmos o design e as partes do nosso site.

Quando temos um documento HTML, tudo aquilo que é exibido na tela, é exibido em forma de caixa, de tal modo que a grande maioria das caixas na HTML podem estar uma dentro da outra. Elas são containers que armazenam conteúdos ou até mesmo outras caixas. Dessa forma, podemos formar hierarquias de caixas.

Considerando que todos os elementos de um site estão dentro de uma caixa, quando formatamos um elemento com o CSS através de seletores, na verdade estamos configurando a caixa que vai estar exibindo esse elemento.

Anatomia das caixas:

  • Conteúdo: Menor unidade de caixa. Por padrão, toda caixa é composta apenas pelo conteúdo e não possui padding, border, outline e margin. Uma exceção é o elemento <body>, que já vem com uma margin de 8px. Todo conteúdo possui uma largura (width) e uma altura (height) e a esse conjunto de propriedades damos o nome de box-size (tamanho da caixa).
  • Padding (acolchoamento): Fica em volta do conteúdo e forma uma lacuna entre o conteúdo e a borda.
  • Border (borda): Envolve o conteúdo e o padding. Podemos traçar uma linha em volta da parte de onteúdo (que define o tamanho da caixa) e chamamos essa linha de borda. Sendo assim, uma borda é uma linha que circunda o seu conteúdo.
  • Outline (contorno): Fica do lado de fora da borda e dentro da margem.
  • Margin: Ao desenvolver um site, eventualmente iremos ter caixas uma embaixo da outra de forma seguida e, se uma dessas caixas estiver configurada com um espaçamento "padding", pode ser que elas fiquem muito próximas uma da outra e acabem "grudando" na borda do elemento de cima. Pensando nisso, podemos criar, além de um espaço interno (padding), um espaçamento externo logo após a borda, chamado de margem.

Vamos criar um exemplo simples para exemplificar todos esses componentes, configurando as propriedades do modelo de caixa de um título <h1>:

<style>
   h1 {
   width: 300px;
   height: 50px;
   background-color: lightgray;
   border-width: 10px;
   border-style: solid;
   border-color: red;
   padding: 20px;
   outline-width: 30px;
   outline-style: solid;
   outline-color: blue;
   margin: 50px;
}
</style>

As linhas width e height configuram o tamanho da caixa (largura e altura, respectivamente) e fará com que ela tenha 300x50 pixels. As linhas que iniciam com border configuram a borda sólida, vermelha e com 10 pixels de espessura. A linha padding cria um espaço interno de preenchimento (da borda para dentro) de 20 pixels e a linha margin cria um expaço externo (da borda para fora) de 50 pixels. As linhas outline configuram o contorno, que faz parte da margem, criando um contorno azul, sólido e com 30 pixels de espessura. O resultado visual do código anterior será:

Olhando de perto, podemos analisar as medidas configuradas no código apresentado. As medidas de height e width (300x50) são medidas apenas pela parte pontilhada do conteúdo.

A border de 10px ficou em vermelho e o outline de 30px ficou em azul. o padding de 20px fica da borda para dentro e a margin de 50px fica da borda para fora.

Nota: As propriedades border-style e outline-style podem ter vários valores: solid (sólida), dotted (pontilhada), dashed (tracejada), double (dupla), groove (3D).

Nota 2: Um recurso muito útil nos navegadores é a opção "inspecionar", que permite ver o código-fonte. Com ela, é possível testar tamanhos de caixas para cada elemento.

Dá pra simplificar?

As configurações de border e outline também possuem shorthands para simplificar o código anterior. A ordem para as duas configurações é sempre a mesma para as duas shorthands: largura (width), estilo (style) e cor (color).

<style>
   h1 {
      border: 10px solid red;
      outline: 30px solid blue;
   }
</style>

Padding e margem personalizados

Todo elemento de caixa possui quatro valores para padding e quatro para margin, sempre nessa ordem: superior (top), direita (right), inferior (bottom), esquerda (left). É importante declarar nessa ordem, pois é necessário ser condizente com a direção e sentido padrão do Modelo de Caixas do DevTools, que segue a direção de um ponteiro de relógio.

Quando colocamos um único valor de dimensão para o preenchimento ou margem, esse mesmo valor é aplicado a todas as direções, por exemplo: padding: 10px;. Mas também podemos fazer códigos como:

<style>
   h1 {
      padding-top: 10px;
      padding-right: 15px;
      padding-bottom: 20px;
      padding-left: 25px;
   }
</style>

Também é possível usar shorthand nesse caso. Essa simplificação só é possível quando as medidas top e bottom forem iguais entre si e o mesmo também ocorrer entre as medidas right e left:

<style>
   h1 {
      padding: 10px 20px;
      margin: 0px 15px;
   }
</style>

Um recurso muito usado é a centralização de blocos. Para centralizar uma caixa, use a seguinte declaração no seletor: margin: auto;. Se quisermos centralizar um elemento de forma automática, mas ainda assim, alterar o margin do sentido de cima (top) ou de baixo (bottom), devemos inserir a seguinte declaração shorthand: margin: 10px auto 10px auto;.

Tipos de caixa:

  • Box-level/block-level: Em uma caixa do tipo "box-level/block-level", o elemento sempre irá se iniciar em uma nova linha e ocupará a largura total da tela ou do elemento onde ele está contido (100% do viewport), sendo que irá pular para a próxima linha antes de continuar o conteúdo posterior e aí sim inserir outros boxes. As caixas desse tipo são isoladas e independentes. Exemplos: <div>, <p>, <main>, <h1>.
  • Inline-level: Não inicia em uma nova linha, e sim no ponto exato onde foi definida, no meio de um parágrafo por exemplo. A largura dele vai ocupar apenas o tamanho relativo ao seu conteúdo, sendo que não irá quebrar a linha ao terminar, continuando o conteúdo de forma direta. Exemplos: <span>, <a>, <button>, <img>.

É possível configurar um elemento block-level para apresentar as propriedades de um elemento inline-level e vice-versa utilizando as propriedades de estilo display: inline e display: block.
