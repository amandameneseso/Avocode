Variáveis e operadores

Variáveis

Variáveis são contêineres para armazenar dados. Podem ser declaradas de 3 maneiras:

1. Usando var

A palavra-chave var foi a forma original de declarar variáveis em JavaScript. Com o tempo, seu uso foi desencorajado em favor de let e const.

Quando você usa var para declarar uma variável dentro de uma função, essa variável só pode ser acessada dentro dessa função (escopo de função). Mas se você usar var fora de qualquer função, a variável se torna global (escopo global), ou seja, ela fica disponível em qualquer lugar do código.

Exemplo dentro de uma função:

function exemplo() {
   var x = 10; // x só existe dentro dessa função
   console.log(x); // Exibe 10
}
console.log(x); // Isso dá erro, pois 'x' só existe dentro da função

Exemplo fora de uma função:

var idade = 20;
var idade = 30; // Pode redeclarar a mesma variável
idade = 40; // Pode reatribuir (alterar) o valor da variável
console.log(idade); // Exibe 40

Utilizando var, as variáveis podem ser globais. Você pode redeclarar a mesma variável várias vezes (o que pode causar confusão e erros). Também pode reatribuí-la (dar um novo valor à variável).

O que é escopo?

Escopo refere-se à área do código onde algo é acessível. Ou seja, escopo de uma variável é o "alcance" que a variável tem dentro de um programa.

1. Escopo de Função: quando você declara uma variável dentro de uma função, essa variável só pode ser acessada dentro dessa função. A variável fica "escondida" fora da função e não pode ser acessada em nenhum outro lugar.

2. Escopo Global: se você declara uma variável fora de qualquer bloco (como funções ou loops), ela se torna global e pode ser acessada de qualquer lugar do código.

3. Escopo de bloco: variáveis declaradas com let ou const dentro de blocos (como { }, if, for) têm escopo de bloco. Elas não podem ser acessadas fora do bloco em que foram declaradas.

4. Escopo léxico: escopo léxico significa que o escopo de uma variável é determinado no momento em que o código é escrito (não em tempo de execução). Funções aninhadas (funções dentro de outras funções) têm acesso às variáveis dos escopos externos onde foram declaradas.

Com o tempo, foi percebido que o var pode causar problemas e confusões no código, principalmente por causa de seu comportamento inesperado. O escopo global torna a variável acessível de qualquer parte do código, o que pode fazer com que qualquer função ou parte do código modifique seu valor acidentalmente. Isso dificulta a manutenção e depuração do código, porque é difícil prever como e onde a variável pode ser alterada. Usar let ou const ajuda a evitar esses problemas, pois elas têm escopo de bloco e são mais restritas ao seu uso.

2. Usando let

let foi introduzido no ECMAScript 2015, como uma alternativa mais segura e moderna a var. Ele tem um escopo de bloco, o que significa que ele só está disponível dentro do bloco em que foi declarado. Além disso, variáveis declaradas com let não podem ser redeclaradas no mesmo escopo.

let idade = 25;
console.log(idade); // Exibe: 25

let idade = 30; // Isso geraria um erro, não pode ser redeclarada
idade = 30; // Pode ser reatribuída, mas não redeclarada
console.log(idade); // Exibe: 30

3. Usando const

const também foi introduzido no ECMAScript 2015. Ele é usado para declarar variáveis constantes. Ou seja, uma vez que uma variável é atribuída com const, seu valor não pode ser reatribuído. Além disso, assim como o let, ele tem escopo de bloco.

const idade = 25;
console.log(idade); // Exibe: 25

idade = 30; // Isso geraria um erro, pois 'const' não permite reatribuição

Nota: Embora você não possa reatribuir o valor de uma variável declarada com const, se essa variável for um objeto ou array, o conteúdo do objeto ou array pode ser modificado internamente.

Resumindo:

  • var podem ser redeclaradas e reatribuídas, o que pode causar problemas de escopo e confusão no código.
  • let tem escopo de bloco e pode ser reatribuída com um novo valor, mas não pode ser redeclarada.
  • const tem escopo de bloco e não pode ser reatribuído nem redeclarado.

Quando usar var, let ou const?

1. Sempre declare variáveis com var, let ou const;

2. Sempre use const se o valor não deve ser alterado;

3. Sempre use const se o tipo não deve ser alterado (Arrays e Objetos)

4. Apenas use let se você não pode usar const;

5. Apenas use var se você estiver trabalhando com navegadores antigos.

Declarando uma variável

Criar uma variável em JavaScript é chamado de "declarar" uma variável. Após a declaração, a variável não tem valor (tecnicamente é undefined). Exemplo:

let cor;
console.log(cor); // Exibe: undefined

Para atribuir um valor para a variável, use o sinal de igual:

cor = 'azul';
console.log(cor); // Exibe: azul

Você também pode atribuir um valor à variável ao declará-la:

let cor = 'azul';
console.log(cor); // Exibe: azul

No exemplo abaixo, criamos uma variável chamada cor e atribuimos o valor "amarelo" para ele. Então, colocamos o valor dentro de um parágrafo HTML com id="texto":

<p id="texto"></p>

<script>
   let cor = "amarelo";
   document.getElementById("texto").innerHTML = cor;
</script>

Nota: É uma boa prática de programação declarar todas as variáveis no início de um script.

Operador de atribuição

Em JavaScript, o sinal de igual (=) é um operador de atribuição, usado para atribuir ou atualizar valores em variáveis. Isso é bem diferente da álgebra, onde o sinal = representa uma igualdade entre dois lados.

Na álgebra:

Uma expressão como:

x = x + 5

é impossível ou sem sentido, porque ela implica que x é igual a x + 5, o que é uma contradição.

Em JavaScript:

A expressão:

x = x + 5

tem um significado claro: o valor atual de x é incrementado em 5, e o novo valor é armazenado de volta na variável x (em outras palavras, x recebe x + 5). Exemplo:

let x = 7; // x começa com o valor 7
x = x + 4; // Soma o valor atual de x (7) com 4, e atribui o resultado (11) de volta a x
console.log(x); // Saída: 11

E se quisermos comparar valores em JavaScript?

Para comparar se dois valores são iguais, usamos == ou ===, que são os operadores de igualdade. A diferença entre esses dois operadores está na forma como esses operadores tratam os tipos de dados durante a comparação.

Diferença entre == e ===:

1. ==: compara apenas o valor das variáveis. Exemplo:

let x = 7;
let y = "7";
console.log(x == y); // Saída: true

2. ===: compara o valor e o tipo das variáveis. Exemplo:

let x = 7;
let y = "7";
console.log(x === y); // Saída: false (número e string são de tipos diferentes)

Tipos de dados

Variáveis podem conter números como 100 e valores de texto como "Maria". Na programação, os valores de texto são chamados de strings.

O JavaScript pode lidar com muitos tipos de dados. Strings são escritas dentro de aspas duplas ou simples. Os números são escritos sem aspas. Se você colocar um número entre aspas, ele será tratado como uma string. Exemplo:

let texto = "Olá, mundo!";
let numero1 = 123;
let numero2 = 45.67;

O que acontece se um número estiver entre aspas?

Se você colocar um número entre aspas, ele será tratado como uma string e não poderá ser usado em cálculos matemáticos diretamente. Exemplo:

let numero = "5";
let soma = numero + 10; // (concatenação de strings, não soma)
console.log(soma); // Saída: 510

Nesse caso, o operador + foi usado para concatenar (juntar) o texto "5" com o número 10, resultando em "510".

Como saber o tipo de dado?

Para saber o tipo de dado, use o operador typeof. Ele retorna o tipo de uma variável ou uma expressão. Exemplo:

let x = 5;
console.log(typeof x); // Saída: number

let y = "Maria";
console.log(typeof y); // Saída: string

Outros tipos de dados:

BigInt: Representa números inteiros muito grandes, além do limite de precisão do tipo Number (que é 253 - 1). Adicione um n ao final do número para criar um BigInt. Exemplo:

let numeroGrande = 12345678901234567890n;
console.log(numeroGrande); // Saída: 12345678901234567890

Boolean (Booleano): Representa um valor lógico que pode ter apenas dois valores: true (verdadeiro) ou false (falso). Exemplo:

let x = 5;
let y = 5;
let z = 6;
(x == y) // Saída: true
(x == z) // Saída: false

Undefined (Indefinido): Uma variável sem um valor tem o valor undefined. O tipo também é undefined. Exemplo:

let x;
console.log(x); // Saída: undefined, tipo: undefined

Nota: valores vazios não são indefinidos. Exemplo:

let x = "";
console.log(x); // Saída: (vazio), tipo: string

Null: Representa a ausência de valor ou um "valor vazio". Difere de undefined, pois null é intencionalmente definido como vazio. Em JavaScript, o tipo de dados de null é um objeto. Isso é um erro histórico no JavaScript, mas foi mantido por questões de compatibilidade com versões antigas. Exemplo:

let x = null;
console.log(x); // Saída: null

Nota: undefined e null são iguais em valor, mas diferentes em tipo:

let x = undefined;
console.log(typeof x); // Saída: undefined

let y = null;
console.log(typeof y); // Saída: object

null == undefined // true
null === undefined // false

Symbol (Símbolo): O Symbol é um tipo de dado usado para criar identificadores únicos e imutáveis. Cada Symbol é único, mesmo que tenha a mesma descrição. Exemplo:

let x = Symbol("meu simbolo");
console.log(x); // Saída: Symbol(meu simbolo)
console.log(typeof x); // Saída: symbol

Object (Objeto): Representa um conjunto de dados estruturados. Objetos podem conter vários tipos de dados, incluindo outros objetos, funções, arrays, etc. São escritos com chaves ({ }), as propriedades são escritas como pares (nome: valor) e separadas por vírgulas. Exemplo:

const pessoa = {
   <nome: "Maria",
   <idade: 30,
   <país: "Brasil"
};
console.log(typeof pessoa); // Saída: object

O objeto (pessoa) no exemplo acima tem 3 propriedades: nome, idade e país.

Nota: Array é um tipo de objeto. É uma estrutura de dados usada para armazenar uma coleção de valores. Ele também pode conter qualquer tipo de dado, como números, strings, objetos, funções e até outros arrays. Um array organiza seus valores em uma ordem indexada, onde cada elemento é associado a um índice numérico que começa do número zero. Exemplo:

let cores = ["Vermelho", "Verde", "Azul"];

console.log(typeof cores); // Saída: "object"

console.log(cores[0]); // Saída: "Vermelho"
console.log(cores[1]); // Saída: "Verde"
console.log(cores[2]); // Saída: "Azul"

Os tipos em JavaScript são dinâmicos. Isso significa que a mesma variável pode ser usada para segurar tipos de dados diferentes:

let x; // Agora x é indefinido
x = 7; // Agora x é um número
x = "Maria"; // Agora x é uma string

Também é possível converter o tipo de uma variável manualmente usando funções ou métodos específicos.

Converter número para string: String()

let a = 123; // Number
let texto = String(a); // Converte para string
console.log(texto); // "123"
console.log(typeof texto); // "string"

Converter string para número: Number()

let texto = "123"; // String
let b = Number(texto); // Converte para number
console.log(b); // 123
console.log(typeof b); // "number"

Comandos para formatar strings

length: Para strings: retorna o número de caracteres na string. Para arrays: retorna o número de elementos no array. Exemplo:

let a = "JavaScript";
console.log(a.length); // 10

let cores = ["Vermelho", "Verde", "Azul"];
console.log(cores.length); // 3

toUpperCase: retorna uma nova string em maiúsculas. Exemplo:

let a = "JavaScript";
console.log(a.toUpperCase()); // JAVASCRIPT

toLowerCase: retorna uma nova string em minúsculas. Exemplo:

let a = "JavaScript";
console.log(a.toLowerCase()); // javascript

Comandos para formatar números

toFixed: limita a quantidade de casas decimais e retorna o valor como uma string. Exemplo:

let a = 3.1415926535;
console.log(a.toFixed(2)); // "3.14"

replace('.', ','): substitui o ponto por uma virgula. Exemplo:

let a = 3.14;
console.log(a.toFixed(2).replace('.', ',')); // "3,14"

Nota: o método replace() não está disponível diretamente para números. O método replace() é um método de strings, então você precisa primeiro converter o número para uma string antes de usá-lo.

Operadores

Assim como na álgebra, em JavaScript, as variáveis também armazenam valores que podem ser usados em cálculos ou operações:

let x = 7;
let y = 3;
let z = x + y;

A partir do exemplo acima, você pode adivinhar que o total calculado é 10.
