Mídias em HTML

Imagens dinâmicas

Para adaptar nosso conteúdo ao tamanho da tela, devemos conhecer as tags <picture> e <source>. A tag <picture> oferece aos desenvolvedores web mais flexibilidade na especificação de recursos de imagem.

O uso mais comum da tag <picture> será para direção de arte em designs responsivos. Em vez de ter uma imagem que é dimensionada para cima ou para baixo com base na largura da viewport, várias imagens podem ser projetadas para preencher melhor a viewport do navegador.

O elemento <picture> contém duas tags: uma ou mais tags <source> e uma tag <img>.

O navegador procurará o primeiro elemento onde a consulta de mídia corresponde à largura da viewport atual e, então, exibirá a imagem adequada (especificada no atributo srcset). O elemento <img> é necessário como o último filho do elemento <picture>, como uma opção de fallback se nenhuma das tags source corresponder.

Para esse exemplo, utilizaremos imagens com 300px, 700px e 1000px de largura. Essas imagens serão carregadas pelo navegador de acordo com o tamanho da janela atual. Para isso, criamos o seguinte código base:


        <picture>
            <img src="imagem-céu-1000.jpg" alt="Imagem flexível">
        </picture>

Inserimos a imagem dentro da tag <picture>, que vai concentrar as outras fontes de imagem. Por padrão, a imagem de 1000px será carregada.

O problema vai começar a surgir quando a janela do navegador chegar perto dos 1000px de largura, pois a foto não vai mais caber lá. Vamos agora adicionar uma linha para resolver esse problema:


        <picture>
            <source media="(max-width: 1050px)"srcset="imagem-céu-700.jpg" type="image/jpg">
            <img src="imagem-céu-1000.jpg" alt="Imagem flexível">
        </picture>

Essas linhas significam que, por padrão, o navegador carregará a imagem grande. Mas se a janela for menor que 1050px, a imagem de 700px (em srcset) será carregada.

Note que a tag <source> possui três atributos:

  • media: indica o tamanho máximo a ser considerado para carregar a imagem indicada no atributo srcset;
  • srcset: configura o nome da imagem que será carregada quando o tamanho indicado for atingido;
  • type: indica o media type da imagem.

Vamos continuar a acrescentar mais um <source> à nossa imagem:


        <picture>
            <source media="(max-width: 750px)" srcset="imagem-céu-300.jpg" type="image/jpg">
            <source media="(max-width: 1050px)"srcset="imagem-céu-700.jpg" type="image/jpg">
            <img src="imagem-céu-1000.jpg" alt="Imagem flexível">
        </picture>

É importate que exista uma ordem crescente entre os <source>, e que o último item dentro de <picture> seja a imagem padrão.

Inserindo áudio

A tag <audio> é usada para incorporar conteúdo sonoro em um documento, como música ou outros fluxos de áudio.

A tag <audio> contém uma ou mais tags com diferentes fontes de áudio. O navegador escolherá a primeira fonte que ele suporta. Caso todas falhem, adicionaremos um parágrafo que permite o download do arquivo. Qualquer texto entre as tags <audio> e </audio> só será exibido em navegadores que não suportam o áudio inserido.

Dica: A tag <audio> funciona similarmente às tags <picture> e <video>. Você configura fontes diferentes, e a primeira fonte que se encaixa nas preferências é a que está sendo usada.

Há três formatos de áudio suportados em HTML: MP3, WAV e OGG.

Principais atributos da tag <audio>:

  • preload: especifica se e como o autor acha que o áudio deve ser carregado quando a página for carregada. Valores:
    • auto (padrão): vai carregar o arquivo de áudio assim que a página for carregada, mesmo que o usuário nunca aperte o play;
    • metadata: carrega apenas as informações sobre o arquivo (tamanho, tempo, etc.)
    • none: o áudio não será pré-carregado, a menos que o usuário consulte o arquivo.
  • controls: especifica que os controles de áudio devem ser exibidos;
  • autoplay: especifica que o áudio iniciará automaticamente;
  • loop: especifica que o áudio será reiniciado sempre que terminar;
  • muted: o áudio será inicializado silenciado;
  • played: adiciona um objeto sinalizando que o áudio foi reproduzido;
  • src: indica a URL do arquivo utilizado.

Exemplo de áudio:

            
            <audio preload="metadata" autoplay controls loop>
                <source src="midia/guanacast-33.mp3" type="audio/mpeg">
                <source src="midia/guanacast-33.ogg" type="audio/ogg">
                <source src="midia/guanacast-33.wav" type="audio/wav">
                <p>nfelizmente seu navegador não consegue reproduzir áudio. <a href="midia/guanacast-33.mp3" download="guanacast-33.mp3" type="audio/mpeg"></a></p>
            </audio>
