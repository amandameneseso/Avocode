Trabalhando com fontes

Categorias de fontes

  • Fonte serifada: possuem hastes e prolongamentos no final das letras. Mais utilizada em títulos e textos para chamar a atenção.
  • Fonte não serifada: Mais utilizada para textos. Transmite modernidade e minimalismo.
  • Fonte monoespaçada: podem ser com ou sem serifa. Possuem a mesma largura para todas as letras.
  • Fonte handwriting/script: imitam a escrita humana. Podem causar cansaço visual.
  • Fonte display: fontes com efeitos visuais.

Como aplicar

Para configurar a família tipográfica que será aplicada, usamo a propriedade "font-family". As seguintes fontes são as melhores fontes seguras para HTML e CSS:

  • Arial (sem serifa)
  • Verdana (sem serifa)
  • Tahoma (sem serifa)
  • Trebuchet MS (sem serifa)
  • Times New Roman (serifa)
  • Geórgia (serifa)
  • Garamond (serif)
  • Courier New (monoespaço)
  • Brush Script MT (cursiva)

Não há fontes 100% seguras para a web. Há sempre uma chance de uma fonte não ser encontrada ou não ser instalada corretamente. Portanto, é muito importante sempre usar fontes de fallback (em sequência). Isso significa que você deve adicionar uma lista de "fontes de backup" semelhantes na propriedade font-family. Se a primeira fonte não funcionar, o navegador tentará a próxima, e a próxima, e assim por diante. Sempre termine a lista com um nome de uma categoria de fontes genérico. Ao aplicar a propriedade "font-family" no VS Code, o programa recomenda uma lista de fontes em sequência em crtl+espaço.

Tamanho das letras

Existem vários tipos de medidas para tamanho de fontes na propriedade "font-size". Os tipos de medidas podem ser divididos em:

  • Medidas absolutas
    • cm (centímetros)
    • mm (milímetros)
    • in (polegadas)
    • px (pixel)
    • pt (ponto)
    • pc (paica)
  • Medidas relativas
    • em (relativo ao tamanho natural da fonte)
    • ex (relativo à altura x da fonte)
    • rem (relativo ao tamanho atual da fonte no ROOT [body ou *])
    • vw (View Width - relativo à largura da viewport/tamanho da tela)
    • vh (View Height - relativo à altura da viewport/tamanho da tela)
    • % (porcentagem)

De todas essas medidas, basicamente vamos trabalhar somente com "em", "px" e talvez o "rem", essa é a recomendação do W3C para sites. O mais recomendado é o "em" porque ele trabalha com porcentagem do tamanho da fonte padrão, isso é bom para o site ser compatível com vários tamanhos de tela.

Para sites, 16px é o tamanho padrão que os navegadores utilizam. Já sobre a medida "em", 1em é igual a 16px. Sendo assim, ao utilizar o "em", use 1em como tamanho normal de fonte.

É possível termos alguns problemas ao utilizar e representar algo com as Medidas Absolutas, pois às vezes o navegador não consegue chegar exatamente na especificação que determinamos. Isso porque existem telas de tamanhos diferentes, como TV's e celulares. Dessa forma, a medida "px" pode sofrer distorções, pois existem telas com diferentes grandezas de pixels.

Peso, estilo e shorthand font

É possível alterar o peso (grossura do caractere) de fontes (que sejam compatíveis). Para alterarmos o peso de uma fonte, usamos a propriedade de estilo "font-weight".Essa propriedade aceita valores/pesos literais e numéricos, como:

Literais: lighter (o mais fino), normal, bold e bolder (o mais grosso).

Numéricos: 100, 200, 300, 400 (normal), 500, 600, 700 (bold), 800 e 900.

Todos esses valores de peso para a propriedade "font-weight" são aceitos para algumas fontes, contanto que a fonte tenha suporte. Fontes como Verdana, Arial e Times não possuem muitas variações de peso.

Outra variação no estilo das fontes é deixar em itálico. Para isso, usamos a propriedade de estilo "font-style" com o valor "italic".

Já para deixar uma fonte sublinhada, usamos a propriedade "text-decoration" com o valor "underline".

Código de exemplo com essas configurações:


        body {
            font-family: sans-serif;
            font-weight: bolder;
            font-size: 1em;
            font-style: italic;}

As formatações de fontes são tão importantes e tão usadas em CSS, que existem atalhos para usá-las. São as chamadas Shorthands (mãozinha). Existe uma shorthand para fontes que é a propriedade "font". No lugar de fazer várias configurações em múltiplas linhas, podemos simplificar tudo de maneira muito simples. Uma shorthand para fontes deve obrigatoriamente seguir essa ordem:

font-style > font-variant > font-weight > font-size > font-family

Devemos inserir os valores dessas propriedades, nessa ordem, ao criar a shorthand. É possível criar shorthands para fontes sem necessariamente usar todas essas propriedades, contanto que a ordem seja respeitada. Exemplo de shorthand para fonte:


        p {
            font: italic bolder 1em sans-serif;}

Alinhamento e recuo da primeira linha

Existem quatro tipos de alinhamentos de texto: left, right, center e justify. A tag utilizada para configurar o alinhamento do texto é "text-align". Já a tag para adicionar um recuo à primeira linha do parágrafo é "text-indent".
